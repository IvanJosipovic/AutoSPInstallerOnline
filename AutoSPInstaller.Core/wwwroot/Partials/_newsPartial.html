Announcing support for SharePoint Server 2019 Preview!

on Wed July 25 2018
We’re pleased to announce that, right behind the heels of the recent announcement of the Public Preview of SharePoint Server 2019, AutoSPInstaller.com now has the ability to help you build an AutoSPInstaller XML configuration to build out a SharePoint 2019 farm.
Make sure you download the most recent version of AutoSPInstaller, then use this site to built out (or validate) your XML configuration.
To be honest, not a whole lot has changed at this point, but at least you’ll be able to target the 2019 version and use some of the new site templates available with the most recent on-premises offering.

Docker Support

We've recived many compaines that AutoSPInstaller Online can't be without internet access or in secure environments due to it being online.
We have packaged up this site into a Docker container that can be used offline at any time. The docker images has Diqus (chat) and telemtry disabled. The source code is availble at https://github.com/IvanJosipovic/AutoSPInstallerOnline

To utilize the image, simply run the following command then navigate to http://localhost
docker run -p 80:80 index.docker.io/autospinstaller/autospinstalleronline

Annoucing SharePoint 2016 support!

on Thu Aug 26 2015

As soon as the Public IT Preview of SharePoint 2016 was announced, people started asking: "Hey when are you gonna update AutoSPInstaller...?" Well luckily, being a Microsoft employee now, I finally have access to some internal non-public releases. I had in fact already been working on updating AutoSPInstaller to support the new on-prem 2016 version. By the time the public preview was announced and made available. I'm happy to announce that, with most of the major compatibility issues ironed out, AutoSPInstaller now works with SharePoint 2016!

Also as an added bonus, AutoSPInstaller also supports installing SharePoint 2016 on Windows Server 2016 Technical Preview 3...!

Having said that, there are some gotchas to be aware of. First, autospinstaller.com has a BETA imlementation for 2016, more validation is required. Second, as with any pre-release software, there are some gotchas and glitches, for example:
  • The installation of SharePoint 2016 itself (with or without AutoSPInstaller) sometimes errors out; most often, re-running the installation will work through this
  • As was the case with previous versions of SharePoint and AutoSPInstaller, sometimes you may see a 'concurrency error' when provisioning certain services or service apps
  • The new SP2016 MinRoles functionality has only been implemented in a basic way so far in AutoSPInstaller; for now you will need to be aware of what services are 'allowed' with each MinRole so as to not accidentally try to provision a service that's not permitted for a certain MinRole (e.g. Search on a DistributedCache server). We hope to implement the 'smarts' behind these service dependencies and relationships within the autospinstaller.com UI soon though.
  • Occasionally, service app creation will hang; you can tell this is happening if it's stuck for a long time on, for example, App Management service app provisioning and the Windows Task Manager shows next to no activity. The fix is to simply go into Central Admin, find the half-provisioned service app, delete it, then re-run AutoSPInstaller
  • Windows Server 2016 TP3 wants to launch URLs with Edge. You may get red errors at the end of the script such as "This app can't be activated when UAC is disabled". These can be ignored as they just relate to the launching of the SharePoint CA/site URLs upon script completion.

Finally, you can (and should) read more about the changes & known issues for SharePoint 2016, and as always you can post a discussion item or report an issue on AutoSPInstaller on the Codeplex project site. Also note that, as always, neither the AutoSPInstaller script/project nor autospinstaller.com are supported by Microsoft.

Cheers

Update

on Thu Jun 18 2015

Hi Everyone,

Welcome to AutoSPInstaller Online, as you may know this site has been live for about a month. We haven't had any formal communication as to what is new or changing in the site. We decided to have a quiet launch so that we can fix any major bugs (there were a bunch, thanks to all that have reported them!) and to make sure the site performed as expected.

New Features:

  • Automatic Schema upgrade from 3.98 > 3.99
    • Simply load your 3.98 XML file and it will be upgrade to 3.99.
    • We decided to update the Schema in order to make it more readable and easier to do side by side comparisons.
    • We plan to have this feature available for all future Schema updates.
  • Alert when leaving the site
    • As the Farm Configuration is only retained in the browsers memory, it will be lost if the user accidentally navigates away from the site.

Planned Features:

  • XML Validation using XSD (XML Schema Definition)
    • There are many scenarios where the XML file is invalid to the point the site will not accept it.
    • One example is boolean values, in the XML world "True" is not a valid value, even though PowerShell accept it.
    • The XSD will return very detailed reason on why the XML fails. For example, ERROR: Element 'ClaimsToWindowsTokenService', attribute 'UpdateAccount': 'True' is not a valid value of the atomic type 'xs:boolean'.
  • Server Rename
    • Allow a Server to be renamed and retain all configuration which was mapped to that server.
  • Predefined Templates
    • Allow the ability to select predefined Farm Templates with recommended topologies, ie 3 Server and 4 Server Farms etc.
  • More Validation
    • There are a lot more places where validation is required.


As always, any feedback is appreciated, we are always looking for ideas on how to make things better.

Stay tuned for more,
AutoSPInstaller Team

Introducing AutoSPInstaller Online

on Thu Jun 18 2015

From Brian Lalancette, creator of AutoSPInstaller.

Since I launched AutoSPInstaller, the PowerShell-based process for installing Microsoft SharePoint Server back in 2010, I've always felt it needed a little something "more". While I and many in the SharePoint community were pretty comfortable hand-editing the input XML file that contained all the farm parameters and settings, a lot of folks weren't necessarily keen on it. In addition, even someone experienced in editing the XML and building SharePoint farms could quite easily miss something or make a simple typo which could lead to pain & confusion at build time (trust me, I know). Not only that, but the process of hand-editing the XML had no rules built into it to help folks adhere to best practices, or even hard dependencies among SharePoint services and service apps. Sure, the script itself helped out with some of this, but it became a bit of a burden to try to capture all these rules within AutoSPInstaller itself plus handling them at run-time isn't optimal IMO.

Then Ivan created AutoSPInstallerGUI - a user-friendly tool that allowed for much easier creation and editing of XML input files for AutoSPInstaller. While this tool was both promising and popular, I personally never got into it much for one reason or another. Maybe hand-editing the XML made me nostalgic for my brief days as a web dev, or maybe Ivan's UI at the time lacked the total control I wanted over the farm config - not sure. Regardless it was an overall well-received tool that certainly helped a lot of folks get their head around using AutoSPInstaller.

Then came the 2014 SharePoint Conference in Las Vegas. There, I finally met Ivan in person, and we got to talking: What if, instead of a UI tool that users needed to download and keep current (bug fixes, new functionality etc.), we provided them with a slick web UI that captured and extended the functionality of AutoSPInstallerGUI, with lots more automation and validation? We felt this would be a huge win for both AutoSPInstaller and the SharePoint community in general. So slowly but surely we (well, mostly Ivan) went to work on it...

A year or so later, we're proud to introduce AutoSPInstaller Online - a slick, responsive web-based interface for creating, editing and validating AutoSPInstaller input XML files. AutoSPInstaller Online is the first such utility to have my full support and ongoing input, so I'm very excited about its potential and value.

Give it a try - we're eager to hear your feedback and plan to continue to improve the site in the coming weeks and months.

Cheers,
Brian
